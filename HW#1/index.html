Основні команди GIT

    Основні команди GIT

  • git config --global user.name blabla - встановити mane користувача
  • git config --global user.email blabal@bla - встановити email користувача
  • git init - створити новий репозиторій
  • git status - Це означає, що файл є новим і git ще не знає, потрібно слідкувати за його змінами, чи краще просто ігнорувати його.
  • git add - додати всі змінені файли до stage
  • git commit - створити новий commit
  • git branch - показує всі гілки
  • git push - відправляє дані локального репозиторію на віддалений
  • git pull - завантажує вміст віддаленого репрезиторію
  • git merge - з'єднує дві гілки в одну
  • git clone - Проект буде клоновано з віддаленого сервера у поточний локальний

Що таке .gitignore файл?

.gitignore служить для вказівки в ньому файлів і папок, які необхідно приховати від системи контролю версій git Як правило, приховують файли або папки, які автоматично генерують свій вміст, або мають конфігураційні параметри, які можуть відрізнятися у тих, хто співпрацює над проектом

  • git revert - створює антикомміт для вказаного комміту, відміняє зміни, внесені в ньому
  • git reset - відміняє зміни файлу
  • git rebase master - підтягнути гілку master в гілку з переміщенням
  • git cherry-pick - Команда git cherry-pick дозволяє забрати один комміт з іншої гілки і вставити його в поточну
  • git stash - призначена для того, щоб помістити поточні зміни, які ми виконали у файлах, в окреме сховище, та повернути файли до вихідного стану
